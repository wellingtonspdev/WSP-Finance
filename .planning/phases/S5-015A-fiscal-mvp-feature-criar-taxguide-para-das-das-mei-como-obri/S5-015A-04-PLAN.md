<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Multipart request to backend | File bytes, filename, MIME, extension, and size are untrusted. |
| Backend to storage provider | Uploaded object must be cleaned up if DB persistence/audit fails. |
| Storage metadata to AuditLog | Object keys and filenames may leak sensitive path/tenant hints and must not be logged. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-S5-015A-04-01 | Tampering | PDF upload validator | mitigate | Enforce MIME, extension, explicit size cap, and signature check where supported. |
| T-S5-015A-04-02 | Information Disclosure | AuditLog upload state | mitigate | Audit hash/flags only; forbid objectKey, filename, raw PDF, base64, CPF/CNPJ/email/name. |
| T-S5-015A-04-03 | Denial of Service | upload size | mitigate | Reject oversized uploads before storage provider call. |
| T-S5-015A-04-SC | Tampering | npm/pip/cargo installs | accept | No package installation is planned. |
</threat_model>
