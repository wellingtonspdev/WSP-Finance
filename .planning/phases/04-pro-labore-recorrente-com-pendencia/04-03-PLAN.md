<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Browser to recurring API | User-entered schedule and confirmation actions cross into backend validation. |
| UI state to financial display | UI presents errors/status without computing authority, tax, or balance mutation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-P4-03-01 | Information disclosure | UI error states | mitigate | Display safe API errors only; do not render raw payloads, PII, account internals, OCR text, or tokens. |
| T-P4-03-02 | Elevation of privilege | Frontend actions | mitigate | Backend remains authoritative; UI hides or disables actions by state but never substitutes for OWNER checks. |
| T-P4-03-03 | Tampering | API payloads | mitigate | Do not send account IDs or tax fields; send explicit workspace IDs, amount, day, description, and pending IDs only. |
| T-P4-03-SC | Tampering | npm/pip/cargo installs | accept | No package installation is planned. |
</threat_model>
