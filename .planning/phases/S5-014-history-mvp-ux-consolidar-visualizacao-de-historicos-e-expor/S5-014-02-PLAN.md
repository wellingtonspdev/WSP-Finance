# S5-014-02 Plan - Frontend Inline Export History

## Objective

Add the S5-014 export-history UI inside the existing extract page. The UI must be collapsed by default, render compact export cards inline below filters, support re-download through the existing download route, and preserve the current export generation modal.

## Scope

### In

- Add frontend API helpers for listing export history and requesting a re-download URL.
- Add a small hook for loading history only when the inline panel is open.
- Add compact export-history cards with expandable user/hash details.
- Add hash truncation plus copy-full-hash action.
- Add `Historico` button beside `Exportar Dominio` in `TransactionHistoryPage`.
- Show empty state with CTA to open `Exportar Dominio`.
- Show in-panel `403` permission message.
- Add focused frontend tests for visibility, empty state, card fields, details, 403, and re-download.

### Out

- No new route/page.
- No backend changes beyond S5-014-01.
- No changes to `ExportService`, Domínio generation, encoding, validation, multi-ERP, NFS-e, TaxGuide, or PGDAS.
- No broad redesign of the extract page.
- No change to the existing `ExportDominioModal` generation flow except wiring empty-state CTA to open it.

## Must Haves

### Truths

- D-01: Export history stays inside `TransactionHistoryPage`.
- D-02: `Historico` button appears next to `Exportar Dominio`.
- D-03: Clicking `Historico` renders inline list below filters and above transaction rows.
- D-04: History is collapsed by default.
- D-05: Each export renders as a compact card.
- D-06: Card always shows period, layout, generated date, status, `recordCount`, `warningsCount`, and download action.
- D-07: Responsible user and hash appear in expandable details.
- D-08: Hash is truncated with a copy-full-hash action.
- D-09: Button visibility uses same gate as `Exportar Dominio`: `BUSINESS` workspace plus `OWNER` or `ACCOUNTANT`.
- D-10: `403` shows `Voce nao tem permissao para ver exportacoes deste workspace.` inside the history panel.
- D-11: Empty state shows simple message plus CTA to `Exportar Dominio`.

### Safety Invariants

- Do not expose or render `objectKey`, bucket, storage path, buffer, base64, or raw file data.
- Use the backend route from S5-014-01 for list and existing workspace download route for re-download.
- Do not fetch history for users who cannot see the `Historico` button.
- Do not convert downloaded export content in the browser; re-download should follow the presigned URL returned by backend.

## Implementation Steps

### 1. Add RED Frontend Tests

Extend `frontend/tests/features/transactions/ExportDominioModal.test.tsx` or create a focused sibling test if the existing file becomes too large.

Required scenarios:
- `Historico` button renders for BUSINESS OWNER.
- `Historico` button renders for BUSINESS ACCOUNTANT.
- `Historico` button does not render for EDITOR, VIEWER, or PERSONAL workspace.
- History panel is not visible by default.
- Clicking `Historico` shows inline history area below filters.
- Empty state shows `Nenhuma exportacao gerada ainda.` and a CTA to `Exportar Dominio`.
- A returned item renders period, layout, generated date, status, `recordCount`, `warningsCount`, and download action.
- Expanding details shows responsible user and truncated hash with copy action.
- `403` from history list shows `Voce nao tem permissao para ver exportacoes deste workspace.`.
- Clicking download calls the frontend helper for `/workspaces/:workspaceId/exports/:archiveId/download` and navigates to the returned URL or otherwise triggers browser download without exposing storage internals in UI.

Mock the export history API helpers in the same module as existing `exportDominio` mocks.

### 2. Add Export History API Helpers

Update `frontend/src/features/transactions/api/exportDominio.ts`.

Suggested additions:
- `listExportHistory(workspaceId: string | number, signal?: AbortSignal): Promise<ExportHistoryResponse>`
- `getExportDownloadUrl(workspaceId: string | number, archiveId: string, signal?: AbortSignal): Promise<ExportDownloadResponse>`

Rules:
- List calls `GET /workspaces/${workspaceId}/exports`.
- Download calls `GET /workspaces/${workspaceId}/exports/${archiveId}/download`.
- Types should live in `frontend/src/features/transactions/types/export.ts` if existing export types are already there.
- DTO must not include storage/internal fields.

### 3. Add Hook

Create `frontend/src/features/transactions/hooks/useExportHistory.ts`.

Rules:
- Use TanStack Query with a stable key, e.g. `['export-history', workspaceId]`.
- Enable query only when `isOpen` and `canExportHistory` are true.
- Map `403` to the exact permission message required by D-10.
- Expose `items`, `isLoading`, `isError`, `errorMessage`, `refetch`, and `downloadArchive`.
- `downloadArchive` should request the download URL then navigate to `url` with `window.location.assign(url)` or another existing safe pattern.

### 4. Add Compact List Component

Create `frontend/src/features/transactions/components/ExportHistoryList.tsx`.

Rules:
- Accept items, loading/error state, empty CTA handler, and download handler as props.
- Render compact cards.
- Show always-visible fields from D-06.
- Use `details`/`summary` or local expand state for user/hash details.
- Hash display should be truncated, e.g. first 8 and last 8 chars, with copy action for full value.
- Use existing dark extract-page visual language; do not introduce a new design system.
- Keep text fitting mobile width.

### 5. Wire TransactionHistoryPage

Update `frontend/src/features/transactions/pages/TransactionHistoryPage.tsx`.

Rules:
- Add `isExportHistoryOpen` state.
- Reuse existing `activeMembership` gate for both buttons.
- Add `Historico` button beside `Exportar Dominio`.
- Render `ExportHistoryList` below filters when open.
- Empty-state CTA sets `isExportModalOpen(true)`.
- Keep transaction filters/list behavior unchanged.

### 6. Preserve Existing Export Modal Behavior

Run existing tests to ensure `ExportDominioModal` still validates and downloads generated exports through `/export/generate` unchanged.

## Acceptance Criteria

- `TransactionHistoryPage` shows `Historico` next to `Exportar Dominio` for eligible BUSINESS OWNER/ACCOUNTANT memberships only.
- History panel is collapsed by default and opens inline below filters.
- Compact cards render all required visible fields.
- Details expansion shows responsible user and hash.
- Hash is truncated but copy action uses full value.
- Empty state has CTA to open `Exportar Dominio`.
- `403` shows the in-panel permission message.
- Re-download uses existing backend download route and does not expose storage internals.
- Existing export modal generation flow remains intact.
