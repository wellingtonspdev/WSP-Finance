<technical_analysis>
## Impacted Modules

- finance-core: `TransactionService`, transaction tests, transaction route/controller contract.
- workspaces/accounts: account lookup for workspace-scoped default resolution.
- rbac-rls/tenant: workspace isolation must be preserved by service and route tests.
- database contracts: no schema migration expected; Prisma model ID/tax/marketplace fields must remain intact.

## Current Flow Diagnosis

`POST /transactions` currently validates `accountId` as required at the controller boundary and the service DTO also requires it. The service then validates explicit ownership with `findByIdAndWorkspace`, computes tax/net values from `Workspace.taxRate`, calculates marketplace final amount from gross/fee/shipping, persists the transaction, updates account balance for paid transactions, writes audit state, and enqueues outbox for expenses.

## Required Technical Change

- Make only manual create `accountId` optional at the controller schema and service DTO boundary.
- In `TransactionService.create`, resolve a `resolvedAccount` before persistence and side effects:
  1. If explicit `accountId` exists, use existing workspace-scoped validation.
  2. If omitted, inspect workspace type and lookup exact seed name:
     - PERSONAL: `Conta PF Principal`
     - BUSINESS: `Conta PJ Principal`
  3. If no exact seed-name account exists, use first `CHECKING` account in the workspace.
  4. If no checking account exists, use first account in the workspace.
  5. If no workspace account exists, fail clearly; do not auto-create.
- Use `resolvedAccount.id` for transaction account connect, paid balance update, audit `newState.accountId`, and `fromAccount`.
- Persist `taxAmount: null` and `netValue: null` for new manual transactions.
- Preserve marketplace fields and `finalAmount = gross - platform fee - shipping` behavior without subtracting tax.

| `backend/src/controllers/TransactionController.ts` | Make `accountId` optional only for create schema; keep other validation intact. |
| `backend/src/services/TransactionService.ts` | Make DTO optional, resolve account, disable tax writes, use resolved account for side effects. |
| `backend/src/repositories/AccountRepository.ts` | Add a narrow workspace-scoped default lookup helper if existing methods are insufficient. |
